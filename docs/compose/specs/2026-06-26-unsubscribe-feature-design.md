# 取消订阅功能设计文档

## [S1] 问题

用户已经订阅了一些播客，但无法管理或取消订阅。需要在主页播客列表和播客详情页面提供取消订阅功能，并支持批量取消订阅。

## [S2] 解决方案概述

采用直接按钮方式：
1. 在主页播客列表中，每个播客项目右侧添加取消订阅图标按钮
2. 在播客详情页面，顶部添加取消订阅按钮
3. 在主页添加“编辑”按钮，进入编辑模式，显示复选框，支持批量取消订阅

## [S3] 架构设计

### UI 层修改
1. **HomeScreen 修改**：
   - 添加编辑模式状态（isEditing）
   - 添加选中的播客集合（selectedPodcasts）
   - 在 TopAppBar 中添加“编辑”按钮，点击后切换编辑模式
   - 在编辑模式下，每个播客项目显示复选框
   - 在非编辑模式下，每个播客项目右侧显示取消订阅图标按钮
   - 添加批量取消订阅按钮，显示在编辑模式下的 TopAppBar 中

2. **PodcastDetailScreen 修改**：
   - 在 TopAppBar 的 actions 中添加取消订阅按钮
   - 添加确认对话框，防止误操作

3. **新增功能**：
   - 批量取消订阅确认对话框
   - 取消订阅成功后的状态更新

### 数据层修改
1. **SubscriptionManager**：
   - 已有 unsubscribe 方法，无需修改
   - 需要添加批量取消订阅方法（或在 UI 层循环调用）

2. **数据库操作**：
   - 已有删除订阅的 SQL 语句，无需修改

## [S4] 组件设计

### 1. HomeScreen 组件

**状态变量**：
- `isEditing: Boolean` - 编辑模式状态
- `selectedPodcasts: Set<String>` - 选中的播客 origin 集合
- `showBatchUnsubscribeDialog: Boolean` - 批量取消订阅确认对话框状态

**UI 组件**：
- **TopAppBar**：
  - 正常模式：标题 + 原有图标 + “编辑”按钮
  - 编辑模式：标题 + “取消”按钮 + “全选”按钮 + “删除选中”按钮

- **播客列表项**：
  - 正常模式：播客信息 + 取消订阅图标按钮
  - 编辑模式：复选框 + 播客信息

- **批量取消订阅对话框**：
  - 确认标题："批量取消订阅"
  - 确认内容："确定要取消订阅选中的 N 个播客吗？"
  - 确认按钮："取消订阅"
  - 取消按钮："取消"

### 2. PodcastDetailScreen 组件

**状态变量**：
- `showUnsubscribeDialog: Boolean` - 取消订阅确认对话框状态

**UI 组件**：
- **TopAppBar actions**：
  - 取消订阅图标按钮（删除图标）

- **取消订阅确认对话框**：
  - 确认标题："取消订阅"
  - 确认内容："确定要取消订阅这个播客吗？"
  - 确认按钮："取消订阅"
  - 取消按钮："取消"

### 3. 交互流程

**单个取消订阅流程**：
1. 用户点击取消订阅按钮
2. 显示确认对话框
3. 用户确认后，调用 SubscriptionManager.unsubscribe()
4. 更新 UI 状态，从列表中移除该播客

**批量取消订阅流程**：
1. 用户点击“编辑”按钮进入编辑模式
2. 用户选中要取消订阅的播客
3. 用户点击“删除选中”按钮
4. 显示批量取消订阅确认对话框
5. 用户确认后，循环调用 SubscriptionManager.unsubscribe()
6. 退出编辑模式，更新 UI 状态

## [S5] 错误处理

### 1. 取消订阅失败处理
- 如果取消订阅失败（例如数据库错误），显示错误提示
- 在 UI 中保持播客项目不变，不移除
- 记录错误日志

### 2. 批量取消订阅部分失败处理
- 如果部分取消订阅失败，显示成功数量和失败数量
- 成功取消订阅的播客从列表中移除
- 失败的播客保留在列表中

### 3. 网络错误处理
- 取消订阅是本地操作，不需要网络
- 如果数据库连接失败，显示通用错误提示
